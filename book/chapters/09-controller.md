# 9. הסבר על הבקר

המחוון דורש: "תמונה, שם, יכולות מעבד וזיכרון". הפרק מציג את לוח
ה־**Raspberry Pi 4 Model B** ששימש בפרויקט, את ה־SoC שלו (Broadcom
BCM2711), את מערכת הזיכרון, ואת מערכת ההפעלה שעליו רץ הקוד. כל
הספציפיקציות מתועדות מתוך ה־datasheet הרשמי של ה־Raspberry Pi
Foundation.

---

## 9.1 שם ודגם

**Raspberry Pi 4 Model B — Rev 1.5 (2022)** עם **8 GB RAM**.

- **יצרן:** Raspberry Pi Ltd. (לשעבר Raspberry Pi Foundation), בריטניה.
- **תאריך השקה של הדגם:** מאי 2019. ה־8GB SKU התווסף במאי 2020.
- **גרסת לוח (Board Revision):** 1.5 — הגרסה היציבה הנוכחית שמטפלת
  בתיקוני USB-C מהגרסה המקורית.
- **שם קוד פנימי במאגר:** `LaserPi` (hostname).

---

## 9.2 תמונת הבקר

![Raspberry Pi 4 Model B מותקן על הבסיס](../photos/raspberry-pi-4b-mounted.jpg)

תמונה של ה־Pi 4B כפי שמותקן על בסיס הפלייווד של הפרויקט (בורגי
M2.5 פליז standoffs בגובה 11 מ"מ). מימין למעלה: כניסת USB-C
לאספקת חשמל. בצד שמאל: שני יציאות HDMI Micro (לא בשימוש בפרויקט;
חיבור גרפי דרך VNC). שתי שורות של USB-A 2.0 + 3.0 — שלוש מהן
פנויות, אחת תפוסה ע"י מצלמת ה־LifeCam. בצד ימין־תחתון: כניסת LAN
1Gbps (לא בשימוש; חיבור רשת דרך Wi-Fi). הכותרת הירוקה ב־top
פינה: ה־GPIO Header של 40 פינים — דרכם מתחבר PCA9685 ומעגל
הלייזר.

## 9.3 יכולות מעבד (SoC: Broadcom BCM2711)

ה־System-on-Chip של ה־Pi 4B הוא **Broadcom BCM2711**, מעבד עם
תכונות הבאות:

| תכונה | ערך |
|---|---|
| ארכיטקטורה | ARMv8-A 64-bit |
| מספר ליבות | 4 (quad-core) |
| מודל ליבה | ARM Cortex-A72 |
| תדר ליבה | 1.5 GHz (לוח revision 1.5; אפשר overclock ל־2.0 GHz) |
| Cache L1 | 32 KB Instruction + 32 KB Data לכל ליבה |
| Cache L2 | 1 MB משותף לכל הליבות |
| הגנת זיכרון | MMU מלא, תמיכה ב־virtualization |
| יחידת floating-point | NEON SIMD לכל ליבה |
| תהליך ייצור | 28 nm |
| צריכת חשמל מקסימלית | ~6.4W בעומס מלא, ~2.7W במצב idle |
| GPU מובנה | VideoCore VI @ 500 MHz (לא בשימוש בפרויקט) |

### 9.3.1 רלוונטיות לפרויקט

הדרישה החישובית של הפרויקט נמדדת ב־§2.7 של הניתוח (תקציב לולאה):

- **`detector.detect()`** (OpenCV pipeline) — ~5–8 ms. הליבות
  הארבע מאפשרות ל־OpenCV (שמשתמש ב־pthread פנימית) לעבד את
  הפעולות הוקטוריות במקביל.
- **`tracker.update()`** (PID) — < 0.1 ms.
- **`camera.capture_frame()`** — תלוי בקצב המצלמה (~33 ms ב־30
  fps), בעיקר wait על V4L2.
- **`servo.move_*` עם `ramp=False`** — ~2 ms ל־I²C transaction.

סך־הכל לפר־פריים: ~26 ms. בתוך תקציב של 33 ms (30 fps). נשארים
~7 ms פנאי לתוספות עתידיות. **ה־CPU של ה־Pi 4B אינו צוואר בקבוק**
במצב הנוכחי.

### 9.3.2 התנהגות בעומס מתמשך

תחת tracking ב־30 fps רציף, טמפרטורת ה־CPU מתייצבת על ~55°C ללא
קירור אקטיבי. ב־CPU של Cortex-A72 הסף ל־thermal throttling הוא
80°C — שולי בטיחות גדולים. גוף קירור אלומיניום פסיבי קטן יכול
להוריד לעוד 5–10°C; לא חובה בפרויקט, רלוונטי רק לסביבות חמות מאוד
או overclocking.

### 9.3.3 למה לא Pi 5 / Pi 3 / Pi Zero

ראו §7.2.1 לניתוח החלופות. בקצרה:

- **Pi 5** היה בחירה לגיטימית — מעבד חזק יותר (Cortex-A76 @ 2.4 GHz)
  — אבל לא הוצדק (Pi 4B מספיק) ולא היה זמין בעת תחילת הפרויקט עם
  שאר הציוד.
- **Pi 3B+** — Cortex-A53 @ 1.4 GHz, 1 GB RAM — היה גורם ל־OpenCV
  לעבוד אבל קרוב לקצה התקציב. לא היה משאיר מקום לפיצ'רים עתידיים.
- **Pi Zero 2W** — quad-core A53 @ 1 GHz, 512 MB RAM. קומפקטי
  ויפהפה, אבל ללא USB-A מובנה (דרוש HUB) וזיכרון לא מספיק
  ל־OpenCV + GUI.

## 9.4 יכולות זיכרון

### 9.4.1 זיכרון עבודה (RAM)

| תכונה | ערך |
|---|---|
| גודל | 8 GB |
| סוג | LPDDR4-3200 SDRAM |
| ערוצי גישה | 32-bit (single channel) |
| תדר אפקטיבי | 3,200 MT/s |
| bandwidth | ~12.8 GB/s |
| מהיר־מעבר | משותף בין CPU ל־GPU |

ה־8 GB נבחרו על־פני 4 GB (זמין במחיר נמוך יותר) מהסיבות הבאות:

1. **OpenCV מקצה zero-copy buffers בגדלים גדולים** — פריימים 640×480
   BGR לוקחים ~900 KB כל אחד; ב־30 fps עם buffering, זה ~30 MB בכל
   רגע נתון.
2. **GUI + tracking + logger במקביל** — `control_panel.py` עם
   `subprocess.Popen` של `test_tracking.py` רץ עם 2 תהליכי Python
   מקבילים. כל אחד מאגר את ה־OpenCV runtime (~150 MB).
3. **שדרוגים עתידיים** (neural detection, depth camera) ידרשו עוד
   זיכרון.
4. **הפרש מחיר זניח** — $10 ההפרש בין 4 GB ל־8 GB לא משמעותי
   בתקציב פרויקט.

### 9.4.2 אחסון משני (microSD)

| תכונה | ערך |
|---|---|
| סוג | microSD UHS-I Class 10 |
| גודל | 64 GB (16 GB מינימום מספיק) |
| יצרן | SanDisk Extreme |
| מהירות קריאה | ~100 MB/s |
| מהירות כתיבה | ~60 MB/s |

האחסון מארח:

- מערכת ההפעלה (~3 GB)
- מאגר הפרויקט (`~/pi/`, כולל `book/`) — ~30 MB
- venv עם תלויות pip (`venv/`) — ~150 MB
- לוגים (`/var/log/`) — ניהול אוטומטי ע"י systemd-journald

הפרויקט אינו רושם וידאו לדיסק — אם פיצ'ר זה יתווסף בעתיד, ייתכן
שיידרש שדרוג ל־SD גדול יותר או הוספת USB SSD.

### 9.4.3 חלוקת זיכרון לזמן ריצה

תוצאות `htop` במצב tracking פעיל:

| תהליך | RAM |
|---|---|
| `control_panel.py` (Python + tkinter) | ~120 MB |
| `test_tracking.py` (Python + cv2 + GUI) | ~180 MB |
| Xorg + VNC + רקע | ~250 MB |
| systemd + שירותים | ~400 MB |
| Cache (file/buffer) | ~600 MB |
| **סה"כ בשימוש** | **~1.55 GB** |
| **פנוי** | **~6.45 GB** |

כלומר רק ~20% מה־8 GB בשימוש בעומס שיא — מרחב גדול לשדרוגים.

## 9.5 ממשקי קלט/פלט (I/O Interfaces)

לוח ה־Pi 4B חושף את הממשקים הבאים, מתוכם הפרויקט משתמש בחלקם:

| ממשק | כמות | בשימוש בפרויקט? | למה? |
|---|:---:|:---:|---|
| **USB-A 2.0** | 2 | ✓ (פורט 1) | מצלמת LifeCam HD-3000 |
| USB-A 3.0 | 2 | — | פנויים לשדרוגים |
| **USB-C (PD)** | 1 | ✓ | אספקת חשמל 5V/3A |
| HDMI Micro | 2 | (פעמים נדירות) | חיבור מסך פיזי כש־VNC לא זמין |
| **GPIO Header** | 40 פינים | ✓ | I²C ל־PCA9685 + GPIO18 לבקרת לייזר |
| **I²C (חומרה)** | 1 ערוץ | ✓ | מסילה Bus 1 (פינים 3, 5) ל־PCA9685 |
| SPI | 2 ערוצים | — | פנוי לשדרוגים (חיישן עומק SPI אפשרי) |
| UART | 1 ערוץ | — | פנוי (debugging דרך serial אם נדרש) |
| Ethernet (RJ45) | 1, 1 Gbps | — | רשת דרך Wi-Fi |
| Wi-Fi 802.11ac | מובנה | ✓ | סנכרון GitHub + SSH + VNC |
| Bluetooth 5.0 | מובנה | — | פנוי |
| MIPI CSI (מצלמה) | 1, 15-pin | — | תכנון ראשוני; הוחלף ב־USB |
| MIPI DSI (תצוגה) | 1, 15-pin | — | פנוי לתצוגה ייעודית בעתיד |

ההיקף הזה הוא חלק מהסיבה לבחירת ה־Pi 4B: כל הממשקים הדרושים
לפרויקט קיימים מובנים בלי תוספות חיצוניות (HAT, מתאם USB, וכו').

## 9.6 חשמל

| תכונה | ערך |
|---|---|
| מתח כניסה | 5.0V (טווח: 4.75–5.25V) |
| זרם כניסה מומלץ | 3A (שיא מותר 6A) |
| מחבר אספקה | USB-C |
| צריכה במצב idle | ~2.7 W |
| צריכה תחת עומס מלא | ~6.4 W |
| צריכה אופיינית במצב הפרויקט (tracking) | ~4.5 W |

הספק USB-C 5V/3A הרשמי של Raspberry Pi מספק בנוחות גם בעת חיבור
מצלמת USB (~500 mA) + PCA9685 לוגי (~50 mA). לכן אין צורך לטעון
את הפינים של ה־5V מ־GPIO לעומסים גדולים — בפרויקט הזה הסרוו מקבלים
חשמל ממקור נפרד (LM2596 מ־12V), לא מה־Pi.

## 9.7 מערכת ההפעלה

| תכונה | ערך |
|---|---|
| הפצה | Raspberry Pi OS (Bookworm, מבוסס Debian 12) |
| ארכיטקטורה | aarch64 (ARM64) — 64-bit |
| Kernel | Linux 6.6.x LTS |
| ספריית C | GNU libc 2.36 |
| Python מובנה | 3.11.2 |
| GUI | LXDE / PIXEL (יחד עם xRDP/VNC לגישה מרחוק) |
| מצב init | systemd 252 |
| ניהול חבילות | apt + dpkg |

הבחירה ב־64-bit (ולא ב־32-bit הוותיק) חשובה לפרויקט:

- `python3-opencv` ב־apt קיים בגרסה מקומפלת ל־ARM64, עם אופטימיזציות
  NEON SIMD פעילות. ב־32-bit הביצועים נמוכים יותר.
- תמיכה רשמית של Raspberry Pi OS עוברת ל־64-bit כברירת מחדל.
- היכולת לטעון יותר מ־4 GB RAM (ה־8 GB של הלוח) דורשת 64-bit.

תיעוד התקנה ראשונית של ה־Pi (תהליך flash של SD, התקנת ספריות,
הקמת cron + VNC) ב־[`docs/setup-pi.md`](../docs/setup-pi.md).
